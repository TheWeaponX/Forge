# 1. Objetivo

O Dashboard é a tela inicial do Forge.

Seu propósito é fornecer ao usuário uma visão clara, rápida e personalizada da sua evolução.

Ele deve responder à pergunta:

"Como estou hoje e qual é o próximo passo?"

---

# 2. Filosofia

O Dashboard não deve ser um mural de informações.

Ele deve destacar apenas o que é relevante naquele momento.

A prioridade é orientar ações, não apenas exibir dados.

# 4. Widgets

## Cabeçalho

- Foto
- Nome
- Nível
- XP
- Streak
- Forge DNA

---

## Treino do Dia

Exibe:

- nome;
- duração estimada;
- modalidade;
- botão "Iniciar".

---

## Objetivos

Lista de metas ativas.

Exemplos:

- Treinar 4 vezes esta semana.
- Beber 2 litros de água hoje.
- Dormir 8 horas.

---

## Estatísticas

- Peso
- Volume semanal
- Tempo treinado
- XP
- Evolução

---

## Skills

Próxima skill recomendada.

Progresso atual.

---

## Desafios

Missões diárias.

Missões semanais.

Eventos.

---

## Conquistas

Últimas badges.

Recordes pessoais.

---

## IA

Resumo do dia.

Exemplos:

"Hoje é um bom dia para focar em membros superiores."

"Você está há três dias sem treinar pernas."

---

## Feed

Publicações de amigos.

Opcional e configurável.

# 5. Fluxos

Ao abrir o aplicativo:

- Atualizar dados locais.
- Sincronizar quando houver internet.
- Exibir widgets disponíveis.
- Destacar notificações importantes.

---

# 6. Personalização

O usuário poderá:

- reordenar widgets;
- ocultar widgets;
- alterar tamanho;
- fixar favoritos.

---

# 7. Inteligência

O Dashboard adapta sua prioridade conforme o contexto.

Exemplos:

- Se houver treino agendado, ele aparece em destaque.
- Se houver um desafio prestes a expirar, ele ganha prioridade.
- Se o usuário estiver offline, o Dashboard continua funcional.

---

# 8. Offline

Todos os dados disponíveis localmente continuam acessíveis.

Widgets dependentes da internet devem indicar claramente quando estão desatualizados.

---

# 9. Critérios de Aceitação

- O Dashboard deve carregar rapidamente.
- Deve funcionar offline.
- Deve ser personalizável.
- Deve apresentar apenas informações relevantes.
- Deve servir como ponto central de navegação do Forge.
