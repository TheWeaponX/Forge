# 1. Visão Geral

O Friends Module é responsável por conectar pessoas dentro do Forge.

Seu objetivo é criar uma comunidade baseada em evolução, motivação e colaboração, e não apenas em números de seguidores.

---

# 2. Objetivos

- Facilitar conexões.
- Incentivar consistência.
- Compartilhar evolução.
- Criar uma comunidade saudável.
- Servir de base para todos os recursos sociais.

---

# 3. Filosofia

O Forge prioriza conexões de qualidade.

O foco não é ganhar seguidores, mas construir uma rede de apoio e evolução.

---

# 4. Modelo Social

O Forge utiliza dois modelos de relacionamento.

## Amigos

Relacionamento bidirecional.

Necessita aceitação.

Permite:

- compartilhar treinos privados;
- participar de desafios exclusivos;
- comparar estatísticas;
- trocar mensagens.

---

## Seguidores

Relacionamento unidirecional.

Não exige aprovação em perfis públicos.

Permite acompanhar publicações e atividades públicas.

---

# 5. Estrutura do Perfil Social

Cada usuário possui:

- foto;
- nome;
- @username;
- bio;
- nível;
- XP;
- streak;
- modalidades favoritas;
- localização aproximada (opcional);
- conquistas públicas.

---

# 6. Amizades

Cada amizade possui:

- data de criação;
- origem do convite;
- status;
- interações recentes;
- desafios em comum.

---

# 7. Seguidores

Lista de:

- seguidores;
- seguindo;
- sugestões.

O usuário poderá remover seguidores ou deixar de seguir pessoas.

---

# 8. Solicitações

Estados possíveis:

- enviada;
- recebida;
- aceita;
- recusada;
- cancelada;
- bloqueada.

---

# 9. Lista de Amigos

Filtros:

- mais ativos;
- recentemente adicionados;
- mesma cidade (opcional);
- mesma academia (futuro);
- mesma modalidade;
- favoritos.

---

# 10. Descoberta de Pessoas

Sugestões baseadas em:

- amigos em comum;
- grupos;
- desafios;
- modalidade;
- objetivos;
- localização aproximada (opcional).

A IA poderá melhorar essas recomendações no futuro.

---

# 11. Interações

Os usuários poderão:

- visualizar perfil;
- enviar convite;
- acompanhar evolução pública;
- comentar publicações;
- reagir;
- compartilhar conquistas;
- desafiar amigos.

---

# 12. Privacidade

Configurações:

- perfil público ou privado;
- aceitar solicitações automaticamente;
- ocultar lista de amigos;
- ocultar seguidores;
- bloquear usuários;
- denunciar comportamento inadequado.

---

# 13. Integração com Outros Módulos

O Friends Module integra-se com:

- Feed
- Groups
- Chat
- Challenges
- Rankings
- Profile
- Achievements
- Timeline

---

# 14. Integração com IA

A IA poderá:

- sugerir novos contatos;
- recomendar grupos;
- indicar parceiros de treino;
- identificar interesses em comum.

---

# 15. Offline

A lista de amigos permanece disponível.

Novas solicitações são sincronizadas quando houver conexão.

---

# 16. Banco de Dados

Entidades principais:

- Friendship
- Follow
- FriendRequest
- UserBlock
- FriendSuggestion

---

# 17. Casos de Erro

- Convite duplicado.
- Usuário bloqueado.
- Perfil privado.
- Falha na sincronização.

O sistema deve informar claramente a causa e permitir nova tentativa quando aplicável.

---

# 18. Critérios de Aceitação

- Solicitações funcionam corretamente.
- Perfis privados respeitam permissões.
- Sugestões são relevantes.
- Sincronização confiável.
- Integração completa com os módulos sociais.

---
